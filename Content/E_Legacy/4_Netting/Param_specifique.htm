Paramétrage spécifique

Avant toute utilisation du module Netting, le paramétrage de certains éléments du menu Paramétrage, dans le panneau Tâches de modules est nécessaire.

Cliquez sur la tâche Paramétrage pour dérouler la liste des six éléments associés.

Conditions de postage de flux

Les opérations de compensation peuvent être affichées dans les autres modules de la suite Sage XRT Treasury.

Le paramétrage permet donc d’affecter les préférences de postage des transactions de compensation d’une convention vers les modules Paiement et Trésorerie.

Vous avez la possibilité de sélectionner des conditions de postage existantes en cliquant sur celles de votre choix dans la liste, ou d'en paramétrer de nouvelles.

Configuration

La commande Configuration ouvre la boîte de dialogue Configuration compensation, composée de deux onglets :

  • Factures
  • Processus de Compensation

Le paramétrage de l'onglet Factures a pour but de définir la base de calcul et la date du traitement de compensation.

Pour sélectionner l'option désirée, cliquez sur le bouton radio de votre choix.

Le paramétrage de l'onglet Processus de Compensation a pour but d'autoriser l'insertion des jours non-ouvrables dans les cycles de compensation, de permettre la suppression des simulations ayant le statut clos et d'appliquer les contrôles du postage.

Pour sélectionner une option, cochez la case appropriée.

Convention de compensation

Le Centre est l’entité la plus importante du module dans la mesure où il administre et lance le traitement de compensation. Les participants sont rattachés au Centre et soumettent leurs données pour que le Centre puisse lancer le traitement de compensation et calculer les règlements correspondants. Les droits d’accès des participants sont définis pour qu’aucune entité ne puisse accéder ou modifier des données sans les droits requis.

Les principales caractéristiques de la convention de compensation doivent être paramétrées via les cinq onglets affichés dans la boîte de dialogue :

  • l’onglet Général, permettant d'identifier la convention
  • l’onglet Entités Internes, permettant d'identifier les participants internes rattachés à un centre de compensation
  • l’onglet Entités Externes, permettant d'identifier les participants externes de la convention
  • l’onglet Devise Factures, permettant de spécifier les devises acceptées dans la convention (facturation)
  • l’onglet Transaction de compensation, permettant de spécifier les codes transaction et le donneur d’ordres utilisé par défaut dans la convention
Un seul et unique Centre de compensation peut être paramétré dans le module.

Cours de compensation

Le processus de compensation repose sur deux conversions de devises :

  • Devise de facturation vers devise de compensation
  • Devise de compensation vers devise de règlement

Il est possible d’utiliser les cours suivants pour paramétrer le processus :

  • Cours Marché (moyen et achat/vente)
  • Cours Budget

Vous avez la possibilité de paramétrer des cours personnalisés pour le calcul de chaque conversion.

Cycles

Les cycles sont les éléments clé du module Netting. Ils déterminent les dates-butoirs de chaque étape du traitement de compensation.

Quatre étapes se produisent au cours d'un cycle :

  • la saisie des factures
  • la simulation de la compensation
  • la validation de la compensation
  • le règlement

La phase de paramétrage est très flexible et les cycles peuvent être créés de manière manuelle ou automatique.

Voir la vidéo

Structure des cycles

Une fois le cycle créé, vous avez la possibilité de définir sa structure interne pour déterminer les périodes des différentes étapes qui le composent.

Le calendrier sur lequel la structure du cycle va se baser, la périodicité et les jalons des diverses étapes doivent être spécifiés.

Il est également possible de définir une structure de cycle à utiliser comme modèle lors du processus de création des cycles. Le modèle est créé comme étant générique, à savoir sans aucun lien avec une convention.
