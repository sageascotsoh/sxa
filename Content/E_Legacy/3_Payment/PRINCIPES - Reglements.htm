Règlements

Règlement est un terme générique pour désigner à la fois les demandes de paiement et les ordres, qu'ils soient complets, incomplets, en attente de transmission ou remis.

Dans le module Paiement, la fonction Transactions offre un accès unique à l'ensemble des transactions.

A partir de ce menu, vous avez également la possibilité d'accéder aux demandes de paiements en cliquant sur l'entrée Demande de paiement

Le menu Transactions ne permet pas d'accéder aux détails d'un règlement. Il n'affiche que le minimum d'informations obligatoires pour l'intégration de l'ordre ou de la demande de paiement à partir d'un fichier comptable.

Une demande devient un ordre dès que son support est connu. Cet ordre évolue ensuite pour devenir progressivement un virement validé, autorisé, etc.

Les différentes catégories de transaction sont identifiées par des codes, créés à partir du menu Eléments communs et du sous-menu Code Opérations. Chaque code transaction est associé à une catégorie de transaction pouvant correspondre à un virement domestique, un virement international, un virement de trésorerie, etc.
Informations minimum

Un règlement doit toujours contenir un minimum d'informations obligatoires :

  • Le type de transaction (virement international, virement domestique, virement commercial, virement de trésorerie, etc).
  • Le payeur
  • Le payé
  • Le RIB à débiter
  • La devise du paiement

La date de règlement prévue, également obligatoire, est renseignée par défaut. Si elle n'est pas appropriée, vous devez penser à la modifier.

Toute demande de règlement ne contenant pas ces éléments obligatoires est rejetée à l'intégration. Le compte à créditer (du tiers) s'affiche par défaut.
Passage d'un règlement de type demande de paiement à une transaction de type virement

Lorsque vous souhaitez créer un ordre dans le module Paiement - par intégration ou par saisie - et que les informations nécessaires ne sont pas renseignées en totalité, vous établissez alors une demande de paiement (contenant le minimum d'informations obligatoires) que vous pouvez consulter et compléter en cliquant sur l'entrée Demandes de paiement.

Pour que votre demande de paiement devienne une transaction de type virement, vous devez fournir les informations supplémentaires qui suivent :

  • Le montant de la transaction
  • Les informations obligatoires et spécifiques à chaque catégorie de transaction
